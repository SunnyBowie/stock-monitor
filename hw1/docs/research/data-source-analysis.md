# 股票盯盘软件数据源可获得性分析

## 结论

课程项目可以实现。建议采用“数据源适配器 + 本地缓存”的方式：演示阶段优先使用 AKShare 的东方财富 A 股实时行情接口，历史日线和财务摘要可使用 AKShare 或 BaoStock；如果接口不稳定，则用内置模拟数据保证原型和课堂验收可运行。

真实商业发布需要采购或授权行情源，不能把公开接口视为长期稳定的商业 SLA。

## 可选数据源

| 数据源 | 可支持能力 | 优点 | 风险与限制 | 建议用途 |
|---|---|---|---|---|
| AKShare | A 股实时行情、历史 K 线、指数、财务等 | Python 开源库，接口丰富，适合课程项目快速验证 | 依赖第三方站点，字段和可用性可能变化；公开接口可能限流 | 首选演示数据源 |
| BaoStock | A 股历史行情、复权因子、基础资料 | 免费、接口稳定度较好，适合历史日 K | 更偏历史数据，不适合秒级实时盯盘 | 日 K 趋势和离线测试 |
| 新浪/腾讯公开行情接口 | 实时价格、涨跌幅、成交量等 | HTTP 接口简单，适合轻量盯盘 | 非正式开放 API，可能跨域、限流或变更 | 备用实时行情适配器 |
| 证券公司/交易所授权数据 | 实时行情、Level-2、合规服务 | 稳定、合规、有 SLA | 成本高，接入复杂 | 商业化版本 |

## 字段可获得性

| 需求字段 | 可获得性 | 说明 |
|---|---|---|
| 股票代码、名称 | 高 | 实时行情接口通常包含代码与名称。 |
| 现价、涨跌额、涨跌幅 | 高 | AKShare 东方财富实时行情或新浪/腾讯行情可返回。 |
| 今开、最高、最低 | 高 | 常见行情字段。 |
| 成交量、成交额 | 高 | 常见行情字段，但单位需要统一。 |
| 分时图 | 中 | 可用实时轮询采样形成简化折线；若需真实逐笔/分钟线，需更稳定数据源。 |
| 日 K 收盘价 | 高 | AKShare/BaoStock 均可支持历史日线。 |
| K 线 OHLC | 中 | 历史开高低收可获得；课程原型可先展示简化蜡烛图，正式实现需处理复权和停牌。 |
| 财务报表摘要 | 中 | AKShare 可获得利润表、资产负债表、现金流量表等公开财务数据；原型阶段只展示近 4 季度营收、净利润和少量指标。 |
| 估值水位 | 中 | 当前 PE 可由行情/财务数据计算，历史分位需要本地缓存或额外历史估值数据支持。 |
| 价格提醒 | 高 | 本地规则计算即可，依赖实时刷新频率。 |

## 推荐技术策略

1. 定义 `IMarketDataProvider` 接口，屏蔽 AKShare、BaoStock、Sina/Tencent 的差异。
2. Windows 客户端通过后端服务或本地 Python/HTTP 服务获取行情，避免 UI 层直接依赖数据源细节。
3. 每 5 到 15 秒刷新自选股列表；分时图按刷新结果追加采样点。
4. 财务报表采用低频缓存策略。营收、净利润、ROE、负债率、经营现金流等字段不需要秒级刷新，可按日或按财报期更新。
5. 本地 SQLite 保存自选股、提醒规则、少量历史采样和财务摘要缓存，断网时展示最后一次行情并标记数据已过期。
6. 验收演示保留 mock provider，确保无网络时原型仍可展示完整流程。
